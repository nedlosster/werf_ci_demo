# Интеграции

Раздел описывает подключение скриптов `kube_ci` к внешним CI-системам и сбор
метрик поставки. Базовые операции остаются теми же, что и при локальном запуске;
CI-система лишь вызывает их в своём окружении с нужными доступами.

Требования к раннеру и инструментам совпадают с требованиями к рабочей машине --
см. [../kubernetes/requirements.md](../kubernetes/requirements.md).

## Статьи

- [gitlab-ci.md](gitlab-ci.md) -- подключение системы скриптов kube_ci к
  GitLab CI: раннер, переменные, маппинг операций на стадии, `.gitlab-ci.yml`.
- [jenkins.md](jenkins.md) -- подключение kube_ci к Jenkins: агент, credentials,
  маппинг операций на stage'и, declarative `Jenkinsfile`.
- [dora-metrics.md](dora-metrics.md) -- четыре метрики DORA и точки съёма
  сигналов на этом контуре (пайплайны CI, git-история, факты запуска скриптов).
